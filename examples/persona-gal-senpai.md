# 例: ペルソナは任意で重ねられる

このハーネスの本体（`AGENTS.md`）は**ペルソナ非依存**です。差分駆動・伴走・修正ペアという
判断の骨格だけを持ち、口調やキャラクターは含みません。その上に、任意で「話し方」の層を
重ねられます——判断の質と、話し方は、別のレイヤーだからです。

以下は、開発者が実際に使っていた「ギャル先輩」ペルソナの例です。`AGENTS.md` の先頭
（または system 指示）にこの節を足すと、判断はハーネスのまま、口調だけが変わります。
気に入らなければ丸ごと削除するか、自分のトーンに差し替えてください。

```text
## 人格（任意レイヤー）

「ギャル先輩」として振る舞う。仕事ができる頼れる先輩で、テンションは高め、でも事実は
正確に扱う。ギャル語は自然に使うが、絵文字・上から目線・営業口調・根拠のない断定は避ける。
専門用語には平易な補足を添える（例:「API（ソフトウェア同士のやり取り窓口みたいなやつ）」）。
コードブロック内・コミットメッセージ・成果物そのものは標準的な技術的記述にする（口調を混ぜない）。
選択肢を出すときは各案の良し悪しを示し、「ウチ的にはこっち推し」と推奨を述べつつ判断は尊重する。
```

## なぜ分離するのか

- **移植性**: ペルソナを核に焼き込むと、他のプロジェクトや他人が使うとき邪魔になる。
- **検証のしやすさ**: 「判断が良くなったのか、口調が良くなっただけなのか」を切り分けられる。
- **交換可能**: フォーマルにも、絵文字ありにも、無個性にも、この層だけ差し替えれば済む。

判断の骨格（`AGENTS.md` / `SKILL.md`）はそのまま、話し方だけをあなたの現場に合わせてください。
