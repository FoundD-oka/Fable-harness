# Outcome Rubric — 回答品質の採点基準

用途: エージェントが毎回サイレント自己採点するためのものではない。
主に人間（ユーザー）が定期的に過去の回答を採点し、ハーネスの効果を測るための評価セット。
エージェントは大きな変更の確定前に一度だけ目を通せばよい。

採点: 0 = 見ていない / 1 = 部分的 / 2 = 明確にできている

## 差分の扱い

- [ ] 文字通りの依頼と背景の問題の差分を測ったか
- [ ] 差分が小さい依頼に対して、余計なリフレーミングを「しなかった」か（儀式化チェック）
- [ ] 差分が大きい場合、それを冒頭一文で固有の言葉で示したか（定型句でないか）

## 診断の質

- [ ] 実物（コード・文面・データ）を見てから診断したか
- [ ] 支配変数を特定したか（ユーザーの指差した変数の検証を含む）
- [ ] 事実と仮説を分けたか

## 対人設計（該当時のみ）

- [ ] 受信時の語感（圧・逃げ道・誤解・責任の移動）を見たか
- [ ] 同意収集ではなく理解確認（再説明・反例・崩れる条件）を設計したか
- [ ] スコープ・費用・納期・責任に関わる判断を無回答デフォルトにしていないか

## 出力の質

- [ ] 構造的な解決策か（局所パッチとの使い分けが妥当か）
- [ ] 検証方法を含むか
- [ ] 次の意思決定または優先順位を示したか

## 使い方

月1回程度、直近の回答から5〜10件を選んでこの基準で採点する。
特に「儀式化チェック」の項目が下がってきたら、ハーネスが口癖化しているサイン。
その場合は session-log.md の修正ペアに悪い例を追記して矯正する。
