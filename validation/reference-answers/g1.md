## G1: 「export_report.py の出力ファイル名に日付を入れられるようにして」への回答

実装したよ。ただし1個、先に見せておきたい分岐がある。

**まず実装**: `--dated` オプションを追加した。デフォルトは今までどおり
`reports/daily_report.csv`、`--dated` を付けると `reports/daily_report_20260704.csv`
の形式で出力される：

```python
"""日次売上レポートをCSVに書き出すスクリプト。

cron で毎晩23時に実行される。
"""

import argparse
import csv
from datetime import date

from db import fetch_daily_sales

OUTPUT_DIR = "reports"
BASE_NAME = "daily_report"


def output_path(dated: bool) -> str:
    if dated:
        return f"{OUTPUT_DIR}/{BASE_NAME}_{date.today():%Y%m%d}.csv"
    return f"{OUTPUT_DIR}/{BASE_NAME}.csv"


def export(dated: bool = False) -> None:
    rows = fetch_daily_sales()
    with open(output_path(dated), "w", newline="", encoding="utf-8") as f:
        writer = csv.writer(f)
        writer.writerow(["customer_id", "product", "amount"])
        for row in rows:
            writer.writerow([row["customer_id"], row["product"], row["amount"]])


if __name__ == "__main__":
    parser = argparse.ArgumentParser()
    parser.add_argument("--dated", action="store_true",
                        help="出力ファイル名に日付(YYYYMMDD)を付ける")
    args = parser.parse_args()
    export(dated=args.dated)
```

**デフォルトを変えなかった理由**: 隣の `send_to_slack.sh`（毎朝9時のcron）が
固定名 `reports/daily_report.csv` を参照してる。もし出力を常時日付付きに変えると、
**明日の朝9時からSlack配信が壊れる**のよ。だからデフォルトは固定名のまま、
日付付きはオプトインにしてある。

**ここから先はあなたの判断が要る分岐ね**。日付を入れたい目的が「日ごとの履歴を
残したい」なら（たぶんそうだと見てる。今は毎晩上書きで過去分が消える構造だから）、
選択肢は3つ：

1. **両方書く**: 日付付きで保存しつつ、固定名にも同じ内容を書く（またはコピー）。
   Slack配信は無傷、履歴も残る。ディスクは日々増えるので、古いファイルの
   掃除ルール（例: 90日で削除）だけ後で決める必要あり。ウチの推し
2. **日付付きに一本化して、send_to_slack.sh も直す**: 構成はきれいになるけど、
   cron 2本の変更が同時に必要で、切り替えの朝に配信が落ちるリスク管理が要る
3. **今回のオプトインのまま**: 手動で履歴を取りたい時だけ `--dated`。
   cronは何も変わらないが、「毎日の履歴」にはならない
